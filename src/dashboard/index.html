Features v8.0

Siempre activos (no se pueden desactivar)
ON Correlation Filter

Antes de ejecutar un trade, compara la pregunta del mercado con tus posiciones abiertas. Si hay >60% de palabras similares, bloquea la compra para no duplicar riesgo.

Activo en: Alert AutoTrader (antes de comprar)

ON Rate Limiting

Controla la velocidad de llamadas a la API de Polymarket. Si recibe error 429 o 500, aplica backoff exponencial automático para no ser bloqueado.

Activo en: Todas las llamadas HTTP a Polymarket

ON Bankroll Tracking

Rastrea tu balance USDC real en Polygon cada 5 min. Calcula ROI, drawdown, win rate y limita el tamaño máximo de apuesta según % del bankroll.

Activo en: Panel Bankroll + verificación antes de cada trade

ON Heatmap

Muestra un mapa de calor con el volumen y cantidad de alertas por categoría de mercado (politics, crypto, sports, etc.). Útil para ver dónde está la actividad.

Activo en: Panel Heatmap (sidebar)

ON Trade Journal

Diario de trading personal. Puedes anotar observaciones, errores y aprendizajes de cada sesión. Se guarda en DB con fecha y hora.

Activo en: Panel Trade Journal (sidebar)

ON Push Notifications

Cuando una alerta tiene score alto (>= umbral configurable), se guarda una notificación en el dashboard con nivel critical/high/medium para revisión rápida.

Activo en: Panel Infraestructura + cada alerta enviada

Análisis Global (afectan todas las alertas)

Alertas Telegram

Como crear tu bot de Telegram:

  1. Abre Telegram y busca @BotFather
  2. Envia el comando /newbot
  3. Elige un nombre para tu bot (ej: "Mi Polymarket Bot")
  4. Elige un username (ej: mi_poly_bot) — debe terminar en "bot"
  5. BotFather te dara un token como: 7123456789:AAH...
  6. Copia ese token y pegalo abajo en "Bot Token"

Como obtener tu Chat ID:

  1. Abre tu bot en Telegram y envia cualquier mensaje (ej: "hola")
  2. Abre en el navegador: https://api.telegram.org/bot<TU_TOKEN>/getUpdates
  3. Busca "chat":{"id":123456789} — ese numero es tu Chat ID
  4. Para grupos: agrega el bot al grupo, envia un mensaje, y repite el paso 2
  5. Puedes poner varios Chat IDs separados por coma para recibir en multiples chats
📡 Sobre WebSocket

El WebSocket es una conexion a nivel sistema con Polymarket para recibir precios en tiempo real (en vez de polling cada 60s). No requiere configuracion por usuario — solo activalo en "Features v8.0" arriba. Reduce la latencia de deteccion de ~60s a menos de 1s.

Parámetros Score

Más alto = menos señales, más certeza

Solo apuesta si quedan menos de X seg

Precio debe estar X ATRs del price_to_beat

% mínimo de ticks en la misma dirección

Cómo funciona el Score

score = distancia_normalizada × factor_tiempo × factor_consistencia

Donde:

distancia = |precio_actual - price_to_beat| / ATR

factor_tiempo = 1 - (tiempo_restante / duración_total)

factor_consistencia = % ticks en dirección dominante

Ejemplo: BTC $200 arriba del PTB, ATR=$120, 90seg restantes de 300seg, 75% ticks up

score = (200/120) × 0.70 × 0.75 = 0.87 ✓ Alta confianza

Parametros

Categorias Excluidas

Rojo = excluida (no se tradea). Verde = permitida. Las alertas de Telegram no se ven afectadas.

Take Profit / Stop Loss

Vende automaticamente la posicion cuando la ganancia o perdida alcanza el porcentaje configurado.

Ej: 30 = vende si sube +30%

0 = desactivado. Ej: 50 = vende si baja -50%

Confirmacion por Price Impact

En vez de comprar al instante, espera X horas y solo entra si el precio se movio a favor (tendencia confirmada).

Tiempo antes de evaluar

Ej: 3 = entra si subio +3%

Descarta si no confirma

Kelly Criterion (Sizing Dinamico)

Calcula el tamaño optimo de cada apuesta basado en probabilidad estimada (score + hit rate). Mas confianza = apuesta mas grande.

0.25 = 1/4 Kelly (conservador). 0.5 = 1/2 Kelly (agresivo)

Trailing Stop Loss

Vende si el precio cae X% desde el maximo alcanzado. Protege ganancias sin salir demasiado temprano.

Ej: 15 = vende si cae 15% desde el precio mas alto

Partial Profit Taking

Vende una fraccion al primer target y deja el resto correr hasta el TP completo o trailing stop.

Ej: 30 = vende fraccion si sube +30%

Ej: 50 = vende la mitad

Auto-Scaling por PNL

Ajusta el tamaño de apuesta automaticamente: ganas mas → apuestas mas, pierdes → apuestas menos.

+10% bet por cada $10 ganados

-20% bet por cada $10 perdidos

Cross-Platform Arbitrage: Polymarket vs Kalshi

Detecta diferencias de precio en el mismo evento entre plataformas. Edge > 3% = oportunidad.

Evento Polymarket Kalshi Edge % Accion Categoria
Click "Escanear" para buscar oportunidades de arbitraje cross-platform

Como funciona:

1. Compara precios del mismo evento en Polymarket y Kalshi

2. Si Polymarket dice 60% YES y Kalshi dice 50% YES, hay un edge de 10%

3. Compras YES barato en Kalshi y vendes YES caro en Polymarket (o viceversa)

4. Requiere cuentas en ambas plataformas. Min edge configurable en Config.
